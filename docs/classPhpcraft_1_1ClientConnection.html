◆ handleEncryptionResponse()

Phpcraft\ClientConnection::handleEncryptionResponse (   $name,
  $private_key 
)

Reads an encryption response packet and authenticates with Mojang. If there is an error, the client is disconnected and false is returned, and on success an array looking like this is returned: 

[
  "id" => "11111111222233334444555555555555",
  "name" => "Notch",
  "properties" => [
    [
      "name" => "textures",
      "value" => "&lt;base64 string&gt;",
      "signature" => "&lt;base64 string; signed data using Yggdrasil's private key&gt;"
    ]
  ]
]

After this, you should call ClientConnection::finishLogin().

Parameters
string$nameThe name the client presented in the Login Start packet.
string$private_keyYour OpenSSL private key resource.
Returns
mixed

◆ handleInitialPacket()

Phpcraft\ClientConnection::handleInitialPacket ( )

Deals with the first packet the client has sent. This function deals with the handshake or legacy list ping packet. Errors will cause the connection to be closed.

Returns
integer Status: 0 = An error occured and the connection has been closed. 1 = Handshake was successfully read; use Connection::$state to see if the client wants to get the status (1) or login to play (2). 2 = A legacy list ping packet has been received.
